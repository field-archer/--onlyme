# 前端环境配置指南

本文档梳理 **本仓库前端** 运行、构建、联调所需的全部环境与配置项（含环境变量、浏览器存储、外部服务）。

---

## 1. 运行与工具链

### 1.1 必要软件

| 项 | 版本要求 | 说明 |
|----|----------|------|
| **Node.js** | ≥ 18（推荐 LTS，如 20.x） | Vite 6 与生态对 Node 有版本要求，低于 16 可能无法正常运行 |
| **npm** | 随 Node 安装即可 | 包管理；亦可用 `pnpm` / `yarn`，需自行对照 lock 策略 |

### 1.2 推荐

- **IDE**：Visual Studio Code + Volar（Vue - Official）
- **浏览器**：Chrome / Edge / Firefox 最新版（需支持 ES 模块、`fetch`、高德 JSAPI 2.0）

## 4. 环境变量（Vite：`VITE_*`）

仅 **`VITE_` 前缀**的变量会注入到前端代码（通过 `import.meta.env` 暴露）。**不要**把真正的密钥提交到 Git；本地使用项目根目录 **`.env`**（该文件已在 `.gitignore` 中忽略）。

### 4.1 变量一览

| 变量名 | 是否必填 | 说明 | 示例（请替换为自有值） |
|--------|----------|------|------------------------|
| `VITE_APP_TITLE` | 建议 | 应用标题（若代码中有引用）；`index.html` 内标题可单独维护 | `森林火灾检测系统` |
| `VITE_AMAP_KEY` | **是**（地图页） | 高德开放平台 **Web 端（JS API）** Key，用于加载地图 | `你的Web端Key` |
| `VITE_AMAP_SECURITY_CODE` | **是**（地图页） | 高德 **安全密钥**（securityJsCode），与 Key 配套；见高德控制台安全设置 | `你的安全密钥` |
| `VITE_API_BASE_URL` | 否 | 后端 **API 根路径**（含 `/api` 后缀，无末尾 `/`）。未设置时默认为 `http://localhost:8000/api` | `http://localhost:8000/api` |

### 4.2 在代码中的使用位置（便于排查）

- `VITE_AMAP_KEY` / `VITE_AMAP_SECURITY_CODE`：`src/composables/useMap.ts`（地图加载与安全配置）
- `VITE_API_BASE_URL`：`src/config/api.ts`（用户登录、火点 CRUD、与 `MainView` 中分析接口同源路径）
- 类型声明：`src/types/env.d.ts`

### 4.3 `.env` 示例（勿提交真实 Key）

在项目根目录新建 `.env`：

```env
VITE_APP_TITLE=森林火灾检测系统
VITE_AMAP_KEY=请填高德Web端Key
VITE_AMAP_SECURITY_CODE=请填高德安全密钥
# 可选；与后端一致即可
# VITE_API_BASE_URL=http://localhost:8000/api
```

本地覆盖可使用 `.env.local`（同样已被 gitignore 相关规则忽略）。

### 4.4 Vite 内置（无需在 `.env` 配置）

| 名称 | 说明 |
|------|------|
| `import.meta.env.MODE` | `development` / `production` |
| `import.meta.env.DEV` / `PROD` | 是否开发/生产 |
| `import.meta.env.BASE_URL` | 应用基础路径（部署在子路径时需配置 `vite.config` 的 `base`） |

## 5. 浏览器端存储（非环境变量，联调时需知）

以下为前端逻辑使用的键名，**非敏感**，主要用于会话与业务衔接：

| 存储 | 键名 | 用途 |
|------|------|------|
| `localStorage` | `forestfire_token` | JWT `access_token` |
| `localStorage` | `forestfire_user` | 当前用户 JSON（如 `id`、`username`） |
| `sessionStorage` | `forestfire_last_fire_count` | 最近一次 **图片/视频分析** 得到的火焰处数（用于地图落库 `fire_count` 与展示一致） |

路由守卫读取 `localStorage` 中的 token 判断 `/map` 是否需要跳转登录（见 `src/router/index.ts`）。

## 6. 后端与网络环境

| 项 | 说明 |
|----|------|
| **API 基地址** | 默认 `http://localhost:8000/api`；与《docs/对接.md》《docs/前端对接-用户与火点标记.md》一致 |
| **跨域** | 开发模式下浏览器会请求后端域名/端口，需后端开启 **CORS** 允许前端来源（如 `http://localhost:5173`） |
| **HTTPS** | 生产环境建议全站 HTTPS；高德 Key 常需配置 **域名白名单** |

## 7. 高德地图（外部服务）

1. 注册 [高德开放平台](https://lbs.amap.com/)，创建应用并添加 **Web 端（JS API）** Key。  
2. 在控制台配置 **安全密钥**（JS API 2.0 要求），填入 `VITE_AMAP_SECURITY_CODE`。  
3. 生产环境建议按官方说明使用 **代理或服务端签名**，避免密钥暴露（见项目内高德 skill / 官方安全文档）。

## 9. 开发注意

- 使用 Vue 3 **Composition API** 与 `<script setup lang="ts">`（与项目现状一致）。  
- 修改环境变量后需 **重启** `npm run dev` 才会生效。  
- 联调顺序建议：后端先启动并可访问 → 配置 `VITE_API_BASE_URL`（若非同默认）→ 再启动前端。

## 10. 常见问题

### 10.1 依赖安装失败

- 检查网络；可尝试 `npm cache clean --force` 后重试 `npm install`。

### 10.2 开发服务器端口占用

- Vite 默认 `5173`；被占用时终端会提示或使用 `--port` 指定端口。

### 10.3 地图白屏 / 鉴权失败

- 核对 `VITE_AMAP_KEY` 与 `VITE_AMAP_SECURITY_CODE` 是否配对、平台是否勾选 JS API 2.0、Referer/域名白名单是否包含当前访问地址。

### 10.4 登录 / 火点接口 401 或跨域错误

- 确认后端已启动、`VITE_API_BASE_URL` 正确；浏览器开发者工具 Network 查看 CORS 与响应 `code`。

### 10.5 类型检查

```bash
npx vue-tsc --noEmit
```
