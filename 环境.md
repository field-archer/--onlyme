# 前端环境配置指南

## 1. 环境要求

### 1.1 必要软件
- **Node.js**: v16.0.0 或更高版本
- **npm**: v7.0.0 或更高版本

### 1.2 推荐工具
- **IDE**: Visual Studio Code
- **浏览器**: Chrome、Firefox、Edge 最新版本

## 2. 项目依赖

### 2.1 核心依赖
- **Vue 3.5.13**: 前端框架
- **Vue Router 4.4.5**: 路由管理

### 2.2 开发依赖
- **TypeScript 5.6.2**: 类型检查
- **Vite 6.0.5**: 构建工具
- **Vue TSC 2.1.10**: Vue 类型检查

## 3. 安装步骤

### 3.1 克隆项目
```bash
git clone <项目仓库地址>
cd <项目目录>
```

### 3.2 安装依赖
```bash
npm install
```

## 4. 运行项目

### 4.1 开发模式
```bash
npm run dev
```

### 4.2 构建生产版本
```bash
npm run build
```

### 4.3 预览生产版本
```bash
npm run preview
```

## 5. 环境变量

### 5.1 后端API地址
在 `src/views/MainView.vue` 文件中，后端API地址配置为：
```javascript
const API_BASE_URL = 'http://localhost:8000/api';
```

如果后端服务运行在不同的地址，请修改此配置。

## 6. 浏览器兼容性

- Chrome: 最新版本
- Firefox: 最新版本
- Edge: 最新版本
- Safari: 最新版本

## 7. 开发注意事项

- 使用 `npm run dev` 启动开发服务器
- 代码遵循 TypeScript 类型规范
- 组件使用 Vue 3 Composition API
- 样式使用 scoped CSS

## 8. 常见问题

### 8.1 依赖安装失败
- 检查网络连接
- 尝试使用 `npm cache clean --force` 清理缓存
- 尝试使用 `npm install --force` 强制安装

### 8.2 开发服务器启动失败
- 检查端口是否被占用
- 检查 Node.js 版本是否符合要求

### 8.3 类型检查错误
- 运行 `npx vue-tsc --noEmit` 检查类型错误
- 根据错误信息修复类型问题
